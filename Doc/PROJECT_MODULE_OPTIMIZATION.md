# 项目模块拆分与优化路线

## 模块划分

### M1. 管理后台认证与访问控制
- 范围：`blog-frontend` 登录、退出、token 存储、路由守卫、未授权跳转
- 目标：把认证逻辑从页面内判断收口成统一入口，减少重复和漏拦截
- 当前状态：本轮处理中

### M2. 后端通用基础层
- 范围：`blog-backend/common`、`config`、异常响应、鉴权基础设施、编码一致性
- 目标：统一返回格式、错误码、过滤器行为和文本编码策略

### M3. 博客内容域
- 范围：文章、分类、标签、文章评论
- 目标：收拢博客内容接口与管理逻辑，减少重复解包和状态分叉

### M4. 社区用户域
- 范围：社区注册登录、个人资料、公开用户主页、头像上传
- 目标：统一用户身份、资料和前后端认证上下文

### M5. 社区内容域
- 范围：节点、帖子、帖子编辑、帖子评论
- 目标：稳定帖子发布、编辑、浏览与评论链路

### M6. 社区治理域
- 范围：审核任务、举报、关键词规则、通知、互动数据
- 目标：把治理流程收成清晰闭环，并为 AI 审核预留稳定边界

### M7. 公开前台体验层
- 范围：`blog-public` 路由、页面加载、API 封装、用户交互体验
- 目标：减少兼容性写法，控制耦合和包体

### M8. 管理后台运营层
- 范围：文章管理、社区帖子管理、审核中心、举报中心、互动概览
- 目标：按业务域收拢页面结构，增强复用和后续可维护性

## 优化顺序

1. M1 管理后台认证与访问控制
2. M2 后端通用基础层
3. M7 公开前台体验层
4. M3 博客内容域
5. M4 社区用户域
6. M5 社区内容域
7. M6 社区治理域
8. M8 管理后台运营层
